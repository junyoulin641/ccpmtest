# Task: Integrate QTimer for Automatic Updates

## Description

Set up a QTimer to automatically call `update_time()` every second, making the clock display update in real-time. This transforms the static display into a live, continuously updating clock.

## Acceptance Criteria

- [ ] QTimer imported from PyQt6.QtCore
- [ ] Timer instance created in `__init__()`
- [ ] Timer configured with 1000ms (1 second) interval
- [ ] Timer connected to `update_time()` slot
- [ ] Timer started in `__init__()`
- [ ] Clock updates automatically every second
- [ ] Time display shows smooth per-second progression
- [ ] No visual flickering or lag during updates

## Technical Details

### Implementation Approach

Add QTimer import and setup in `clock_app.py`:

```python
from PyQt6.QtCore import QTimer

# Inside ClockWindow.__init__(), after label setup:

# Set up timer to update every second
self.timer = QTimer()
self.timer.timeout.connect(self.update_time)
self.timer.start(1000)  # Update every 1000ms (1 second)

# Initial update
self.update_time()
```

### Key Components

1. **QTimer Instance**:
   - Store as `self.timer` for lifecycle management
   - Part of Qt event loop (no manual threading needed)

2. **Timer Configuration**:
   - Interval: 1000 milliseconds (exactly 1 second)
   - Connected to: `self.update_time` method
   - Started immediately in `__init__()`

3. **Signal/Slot Connection**:
   - `timer.timeout` signal emitted every interval
   - Connected to `update_time()` slot
   - Qt handles the connection and event dispatch

4. **Event Loop**:
   - Timer integrated with QApplication.exec()
   - Automatic and efficient (event-driven, not polling)
   - Near-zero CPU usage when idle

### Key Considerations

1. **Accuracy**:
   - Each tick calls `datetime.now()` (no cumulative drift)
   - Timer triggers every ~1000ms (system-dependent precision)
   - Display updates within 100ms of actual second change

2. **Performance**:
   - QTimer is lightweight and event-driven
   - CPU usage should be < 1% when idle
   - No manual threading or sleep loops needed

3. **Lifecycle**:
   - Timer starts when window initializes
   - Stops automatically when window closes
   - No manual cleanup required (Qt handles it)
