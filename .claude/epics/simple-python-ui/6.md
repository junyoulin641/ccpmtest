# Task: Manual testing on Windows and macOS

## Description

Perform comprehensive manual testing of the Simple Python UI application on both Windows and macOS platforms. Verify all functional requirements, test edge cases, check cross-platform compatibility, and ensure all acceptance criteria from the PRD are met.

## Acceptance Criteria

- [ ] Application launches successfully on Windows
- [ ] Application launches successfully on macOS
- [ ] All UI components render correctly on both platforms
- [ ] Text processing (uppercase conversion) works correctly
- [ ] Empty input validation works on both platforms
- [ ] Error handling works for various error scenarios
- [ ] Status bar updates correctly through all states
- [ ] Window resizing works properly on both platforms
- [ ] Long text inputs are handled correctly
- [ ] Special characters are processed correctly
- [ ] No crashes or unhandled exceptions occur
- [ ] All PRD acceptance criteria are verified

## Technical Details

**Test Cases:**

**TC1: Normal Operation**
1. Launch application: `python main.py`
2. Enter text: "hello world"
3. Click Execute button
4. Verify: Result shows "HELLO WORLD"
5. Verify: Status shows "✅ Complete"

**TC2: Empty Input Validation**
1. Launch application
2. Click Execute without entering text
3. Verify: Status shows "⚠️ Please enter some text first"
4. Verify: No result displayed

**TC3: Long Text Handling**
1. Enter 1000+ character text
2. Click Execute
3. Verify: Processing completes in <100ms
4. Verify: Result displays correctly (scrollable if needed)

**TC4: Special Characters**
1. Enter text with special chars: "Hello @#$% 123 世界"
2. Click Execute
3. Verify: All characters processed correctly
4. Verify: No encoding errors

**TC5: Window Resizing**
1. Resize window to minimum size
2. Verify: All widgets remain visible and usable
3. Resize window to maximum
4. Verify: Layout adapts appropriately

**TC6: Multiple Operations**
1. Process text: "first"
2. Clear input, enter: "second"
3. Click Execute
4. Verify: Previous result is cleared
5. Verify: New result displays correctly

**TC7: Error Simulation**
1. Modify `processor.py` to raise an exception
2. Process text
3. Verify: Error message displays in status bar
4. Verify: Application doesn't crash
5. Restore `processor.py`

**Platform-Specific Checks:**

**Windows:**
- Font rendering is clear
- Button clicks are responsive
- Window decorations are correct
- No console window appears (if using pythonw.exe)

**macOS:**
- Retina display rendering is sharp
- Native look and feel
- Menu bar integration (if applicable)
- Keyboard shortcuts work (if implemented)

**Cross-Platform:**
- Layout is consistent
- Padding and spacing are similar
- Colors and fonts are readable
- No platform-specific bugs

**Performance Checks:**
- Application launch time < 2 seconds
- Text processing < 100ms for typical inputs
- UI remains responsive during processing
- No memory leaks during extended use
