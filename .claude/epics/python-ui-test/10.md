# Task: Main Window and Navigation Framework

## Description

Implement the main application window (QMainWindow) with navigation infrastructure including QStackedWidget for screen switching, sidebar navigation with buttons, menu bar, toolbar, and status bar. This provides the foundation for all subsequent screen implementations.

## Acceptance Criteria

- [ ] MainWindow class created in ui/main_window.py inheriting from QMainWindow
- [ ] QStackedWidget implemented for managing multiple screens
- [ ] Navigation sidebar with 6 buttons (Dashboard, Import, Table, Visualize, Settings, Export)
- [ ] QMenuBar with File, Edit, View, Tools, Help menus
- [ ] QToolBar with quick-access navigation buttons
- [ ] QStatusBar showing application status messages
- [ ] Screen switching works between navigation buttons
- [ ] Active screen highlighted in sidebar
- [ ] Keyboard shortcuts implemented (Ctrl+1 through Ctrl+6)

## Technical Details

**Implementation Approach:**

Create `ui/main_window.py` with MainWindow class:
- Use QMainWindow as base class
- Create QStackedWidget as central widget
- Add QDockWidget or QFrame for left sidebar
- Create QPushButton for each navigation item
- Connect button signals to screen switching slots

**Layout Structure:**
```
QMainWindow
├── QMenuBar (top)
├── QToolBar (below menu)
├── Central Widget (QWidget)
│   ├── QHBoxLayout
│   │   ├── Sidebar (QFrame with QVBoxLayout)
│   │   │   ├── Dashboard Button
│   │   │   ├── Import Button
│   │   │   ├── Table Button
│   │   │   ├── Visualize Button
│   │   │   ├── Settings Button
│   │   │   └── Export Button
│   │   └── QStackedWidget (screen container)
└── QStatusBar (bottom)
```

**Navigation Logic:**
- Each button connected to `switch_screen(index)` method
- QStackedWidget.setCurrentIndex() to change screens
- Update button styles to highlight active screen
- Emit statusbar messages on screen change

**Menu Structure:**
- File: New, Open, Save, Save As, Exit
- Edit: Preferences (placeholder)
- View: Full Screen, Zoom In, Zoom Out
- Tools: Refresh Data (placeholder)
- Help: About, Documentation

**Keyboard Shortcuts:**
- Ctrl+1: Dashboard
- Ctrl+2: Import
- Ctrl+3: Table
- Ctrl+4: Visualize
- Ctrl+5: Settings
- Ctrl+6: Export
- Use QAction with setShortcut()

**Placeholder Screens:**
- Create 6 simple QWidget subclasses as placeholders
- Each shows a QLabel with screen name
- Will be replaced in subsequent tasks

**Styling:**
- Active button: different background color
- Inactive button: default style
- Use QSS (Qt Style Sheets) for consistent appearance
- Create basic styles in resources/styles/main.qss

**Key Considerations:**
- Use signals/slots for loose coupling
- Separate navigation logic from screen logic
- Make it easy to add/remove screens later
- Ensure keyboard navigation works when buttons have focus

**Files to Create:**
- `ui/main_window.py` - Main window class
- `ui/placeholder_screens.py` - Temporary placeholder widgets
- `resources/styles/main.qss` - Basic styling

**Files to Modify:**
- `main.py` - Import and instantiate MainWindow instead of basic QMainWindow
