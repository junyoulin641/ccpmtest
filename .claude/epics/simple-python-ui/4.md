# Task: Integrate UI with processing and add validation

## Description

Connect the tkinter UI to the processing logic by implementing the button click handler. Add input validation to check for empty inputs, implement error handling with try-except blocks, and update the status bar to reflect the current state (Idle/Processing/Complete/Error). Display results in the result area.

## Acceptance Criteria

- [ ] Execute button is connected to event handler
- [ ] Input validation checks for empty input
- [ ] Empty input shows error: "⚠️ Please enter some text first"
- [ ] Valid input is passed to `processor.process_text()`
- [ ] Processing results are displayed in result area
- [ ] Status bar updates to "⏳ Processing..." during processing
- [ ] Status bar updates to "✅ Complete" on success
- [ ] Processing errors are caught and displayed as "❌ Error: [message]"
- [ ] Previous results are cleared before new processing
- [ ] Application doesn't crash on any user input

## Technical Details

**Implementation Approach:**

1. **Create event handler function:**
```python
def on_execute():
    """Handle execute button click"""
    # Clear previous results
    result_text.config(state='normal')
    result_text.delete('1.0', tk.END)

    # Get input
    input_value = input_entry.get().strip()

    # Validate input
    if not input_value:
        status_bar.config(text="⚠️ Please enter some text first")
        result_text.config(state='disabled')
        return

    # Update status to processing
    status_bar.config(text="⏳ Processing...")
    root.update_idletasks()  # Force UI update

    try:
        # Process text
        result = process_text(input_value)

        # Display result
        result_text.insert('1.0', result)
        status_bar.config(text="✅ Complete")

    except Exception as e:
        # Display error
        error_msg = f"❌ Error: {str(e)}"
        status_bar.config(text=error_msg)
        result_text.insert('1.0', f"Processing failed: {str(e)}")

    finally:
        result_text.config(state='disabled')
```

2. **Bind handler to button:**
```python
execute_btn.config(command=on_execute)
```

3. **Optional: Handle Enter key in input field:**
```python
input_entry.bind('<Return>', lambda e: on_execute())
```

**Data Flow:**
```
User clicks Execute
    ↓
Get input from Entry widget
    ↓
Validate (not empty)
    ↓
Status → "Processing..."
    ↓
Call processor.process_text()
    ↓
Display result in Text widget
    ↓
Status → "Complete" or "Error"
```

**Error Handling:**
- Empty input: Show warning in status bar
- Processing exception: Catch and display user-friendly error
- Always disable result Text widget after update (read-only)
